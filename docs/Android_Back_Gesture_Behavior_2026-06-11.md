# Android 全屏返回手势行为

## 目标

真机上不能只依赖左上角返回按钮。Android 全屏手势和系统返回键必须符合常见 App 习惯。

## MVP 行为

1. 二级页面：系统返回回到上一级页面。
2. 一级 Tab：发现、地图、消息、我的页面系统返回先回到宠物首页。
3. 首页/登录/权限入口/未建档入口：第一次系统返回显示 Toast「再按一次退出灵伴」，1.8 秒内第二次返回退出 App。
4. 弹窗或地图样式面板打开时：系统返回先关闭当前浮层，不直接退出页面。

## 备注

iOS 的左滑返回需要后续接原生导航栈后统一处理。当前 React Native MVP 先保证 Android 真机全屏手势可用。
