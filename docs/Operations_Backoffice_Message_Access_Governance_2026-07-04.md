# 运营后台：私信查看治理策略

日期：2026-07-04

## 目标

把“运营能否查看用户私信、查看多少、谁负责、保留多久”从待澄清问题落成可执行策略，避免关系消息排查和举报处理停留在口头约定。

## 当前策略

- 不开放任意私信全文检索。
- 不开放全量历史会话查看。
- 仅允许在举报处理或关系消息排查中查看最近上下文窗口。
- 默认上下文窗口为最近 20 条，可在配置中心调整为 5-50 条。
- 默认必须填写查看原因。
- 单 admin 阶段采用“带审计的自审批”：查看人、查看原因、会话 ID、双方手机号、窗口条数、策略快照和保留到期标记都会写入审计。
- 审计保留标记默认 180 天，可配置为 7-365 天。

## 后台入口

### 配置中心

配置键：`social.messageAccess`

字段：

- `contextWindowLimit`：最近消息窗口条数，5-50。
- `requireReason`：查看上下文是否必须填写原因。
- `retentionDays`：审计保留标记天数，7-365。
- `fullConversationSearchEnabled`：固定为 `false`，当前不开放全文检索。

配置中心会把这组策略纳入：

- 配置快照。
- 配置变更摘要。
- 配置联动体检。
- 风险变更：关闭原因必填为 P1，调整窗口和保留标记为 P2。

### 关系消息

`GET /admin/social-relations` 返回 `messageAccessPolicy`，页面会展示：

- 上下文窗口条数。
- 查看原因是否必填。
- 全文检索是否开放。
- 审计保留标记天数。

`POST /admin/social-relations/{relationId}/message-context` 会按策略强制裁剪窗口，即使请求传入更大的 `limit`，也不会超过配置值。

## 移动端联动

私信查看本身不下发给移动端。

但治理动作会影响移动端：

- 后台隐藏违规私信后，双方 `/conversations/{conversationId}/messages` 不再返回该消息。
- `/conversations` 会重新计算摘要，避免会话列表继续露出被隐藏文本。
- 举报中心和关系消息页复用同一隐藏工具函数，移动端表现一致。

## 仍未开放

- 任意关键词搜索私信全文。
- 全量会话历史展开。
- 后台新增、改写或补发用户私信。
- 隐藏私信后自动创建处罚；当前仍应从举报中心、内容安全任务或用户处罚页继续处理账号限制。

## 后续可升级

- 多管理员后，把高敏私信查看升级为双人审批。
- 按举报/内容安全任务自动生成查看申请单。
- 针对未成年人、涉医疗、涉诈骗等高风险会话启用更短窗口或更严格审批。
