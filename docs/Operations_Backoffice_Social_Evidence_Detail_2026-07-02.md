# 运营后台宠友圈证据详情入口
日期：2026-07-02

## 背景

宠友圈后台已经支持动态/评论列表、举报处理、内容隐藏/删除和作者快捷处罚，但运营在处理内容时仍需要一个聚合视角，能在同一处看到内容原文、图片、举报、评论、作者当前处罚和近期后台动作。

本次补齐的是“证据详情入口”，用于降低误处理和漏处理风险。

## 本次实现

新增后台接口：

- `POST /admin/social/posts/{postId}/evidence`
- `POST /admin/social/comments/{commentId}/evidence`

查看证据必须填写原因，空原因会返回：

- `ADMIN_SOCIAL_EVIDENCE_REASON_REQUIRED`

后台宠友圈页面新增能力：

- 动态行增加“证据”按钮。
- 评论行增加“证据”按钮。
- 点击后在当前页面展开同页证据面板，不跳出当前列表上下文。
- 面板展示内容快照、图片、状态、创建时间、查看原因、举报、直接关联处罚、作者当前生效限制和近期审计日志。

## 证据范围

动态证据详情聚合：

- 动态 ID、作者、宠物名、正文、图片、状态和发布时间。
- 该动态下的评论数、点赞数和图片数。
- 该动态下的评论列表。
- 该动态自身举报。
- 该动态下评论产生的举报。
- 作者当前生效处罚。
- 与该动态或子评论直接关联的处罚。
- 相关后台审计日志。

评论证据详情聚合：

- 评论 ID、作者、正文、状态和发布时间。
- 父级动态摘要。
- 该评论举报。
- 评论作者当前生效处罚。
- 与该评论直接关联的处罚。
- 相关后台审计日志。

## 审计

每次查看证据都会写入审计日志：

- `social.post.evidence.view`
- `social.comment.evidence.view`

审计记录会保存：

- 管理员。
- 目标类型和目标 ID。
- 查看原因。
- 目标作者手机号。
- 举报数、评论数、图片数等摘要字段。

## 移动端联动

该能力不是独立后台数据：

- 动态、评论、点赞和举报来自移动端真实业务接口。
- 隐藏、删除和处罚仍然复用现有移动端可见性和账号限制体系。
- 作者处罚后，移动端发布小事、评论、点赞、打招呼、约遛、私信、地点投稿和地点点评等写接口会实时受禁言/冻结/封禁影响。
- 证据详情展示的状态会反映当前移动端内容是否仍可见。

## 验证

新增烟测：

```bash
node scripts/smoke-social-evidence-detail.cjs
```

脚本会创建真实用户、小事、评论和举报，然后通过 admin 接口查看动态证据和评论证据，并确认审计日志已写入。

## 仍未覆盖

- 独立证据快照表。
- 图片类证据长期归档。
- 水印化取证文件。
- 双人复核或质检抽样队列。
