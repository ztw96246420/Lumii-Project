# 宠物资料后台修正

日期：2026-07-03

## 目标

补齐运营后台对宠物档案的基础修正能力，让后台不只可以清理违规媒体，也可以在用户反馈、客服排查或明显错误时协助修正宠物资料。

## 后台能力

- `GET /admin/pets/{petId}`
  - 返回宠物详情、主人摘要和列表行口径。
- `PATCH /admin/pets/{petId}`
  - 支持修正：昵称、类型、品种、性别、生日、体重。
  - 必须提交 `reason`。
  - 复用移动端 `parsePetProfilePayload` 校验规则。
  - 复用宠物资料内容安全检查，风险内容不会绕过审核。
  - 写入 `adminAuditLogs`，action 为 `pet.profile.update`。
  - 给用户写入站内通知 `宠物资料已修正`。

## 移动端联动

- 后台直接更新同一份 `users.pets`。
- 移动端 `/pets`、`/me`、首页宠物卡片、宠物日历、AI 对话上下文和宠友圈宠物信息下一次刷新都会读取新值。
- 用户会在通知中心看到平台修正说明，避免资料突然变化但没有解释。

## 当前边界

- 普通头像和宠友圈封面已由独立媒体替换能力承接；仍不支持后台替换 AI 灵伴形象结果图，避免伪造 AI 任务产物。
- 不支持合并重复宠物档案；合并需要迁移宠物日历、AI 任务、宠友圈、约遛/会话等引用，生产期需要更细权限和迁移审计。
- 不支持后台直接删除宠物档案；删除会影响多条业务链路，仍保留在后续治理范围。

## 验收

- `scripts/smoke-admin-pet-profile-edit.cjs`
  - 用户创建宠物。
  - 后台无原因修正被拒绝。
  - 后台带原因修正资料成功。
  - 移动端 `/pets` 读到新资料。
  - 用户通知中心收到修正通知。
  - 审计日志存在 `pet.profile.update`。
