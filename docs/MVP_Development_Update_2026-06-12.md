# MVP Development Update 2026-06-12

## 本次进展

- 设备推送 token 上报校验补强：`POST /devices/push-token` 现在只接受 `token`、`platform`、`deviceId` 三个字段；`token` 必填且限制基础长度和控制字符，`platform` 必须是 `android/ios/web`，`deviceId` 可选但限制基础长度和控制字符。非法输入返回 `PUSH_DEVICE_INVALID`，不再把未知平台静默兜底成 Android。mock API 同步该规则，不新增页面。
- 文档口径同步：接口契约、支持清单和 Figma 缺失清单已把设备推送 token 字段兜底校验标记为 MVP 已实现；生产厂商通道、通知模板、送达回执仍作为后续推送专项。
- 执行优先级调整：后续在“不新增页面、不接新外部模型”的前提下，优先推进真机可感知的业务闭环和状态一致性，例如二次登录状态恢复、建档后首页/健康/发现数据同步、AI 对话写入业务动作后的回显；灰产词、复杂内容风控等安全兜底暂不作为靠前优先级。
- 当前宠物业务状态一致性修复：App 现在会把 `activePet` 变化同步回 `session.account.activePet`，覆盖编辑档案、保存形象、手动/AI 记录体重、健康摘要刷新等路径；同时 `loadCommonData` 改为只在登录 token 变化时做全量拉取，避免账号快照更新导致重复全量刷新。不新增页面。
- 真实账号宠物状态兜底收紧：HTTP 真后端模式下，登录恢复和 `/me` 刷新只信账号快照与真实宠物列表，不再用 mock 宠物补当前宠物；没有宠物时会回到“未添加宠物”，宠物首页、健康、AI 对话、上传/生成、上传未检测到宠物等宠物必需页面也增加现有空态守卫，避免无宠物账号看到旧宠物或默认演示宠物。不新增页面。
- AI 对话业务动作同步补强：AI 回复携带 `updatedPet`、`createdWeight`、`createdMemo`、`updatedVaccine`、`vaccineReminderIds` 或 `medicalAlert` 时，App 会统一同步宠物档案、体重记录、健康备忘、疫苗计划、提醒开关、健康摘要和通知中心，并用现有 toast 提示“已同步”。同时补齐本地疫苗提醒列表减少时的系统通知取消，避免 AI 对话关闭提醒后真机本地通知残留。不新增页面。
- AI 对话动作防重复补强：发送消息和反馈“像它/不像它”增加本地 ref 锁，减少真机快速连点造成的重复发送或重复反馈；退出账号时同步清理反馈保存中和回复中状态。不新增页面。
- 健康记录保存防重复补强：体重记录、健康首页快捷备忘、健康备忘页新增和今日小事发布增加本地 ref 锁，减少真机连点造成的重复提交窗口；快捷备忘保存成功后会同时清空标题和内容，避免下一条误用旧标题。不新增页面。
- 新用户建档/电子形象保存后的宠物维度刷新补强：新建宠物成功后会刷新健康摘要、体重、疫苗/驱虫、提醒开关、健康备忘和 AI 用量；保存电子形象成功后会同步当前宠物、刷新宠物维度数据并清理上传/生成草稿，避免回首页或再次上传时读取旧任务状态。不新增页面。
- 社交发现动作回显补强：发送打招呼、接受/婉拒招呼、发送约遛邀请现在会使用现有 loading/disabled 状态防重复点击，并在成功后刷新真实招呼请求、会话列表和通知中心，不再依赖本地临时假通知；退出账号也会清理社交发送中状态。不新增页面。
- 招呼请求互斥保护：同一条招呼请求的“接受”和“婉拒”现在会互斥处理中，操作前会确认请求仍在当前列表里；请求已刷新或消失时提示返回消息页刷新，避免同一请求并发接受/拒绝。不新增页面。
- 执行优先级校准：下一阶段优先补齐已有页面内的业务闭环和状态一致性，例如二次进入、数据回显、刷新、回退、防重复提交和失败恢复；内容安全细则、灰产词扩展、复杂风控规则暂不抢 MVP 主线，只保留后续接口口子。
- 执行优先级再次收敛：在不新增页面、不接新外部接口/模型的前提下，后续优先处理现有页面里的业务状态闭环，例如误发保护、选中对象同步、切号清理、刷新后回显和失败恢复；灰产词库扩展、复杂内容审核和安全教育卡片暂时降级。
- 建档/上传/AI 形象生成主链路状态保护补强：宠物建档保存、相册/相机上传、启动生成、生成轮询、重新生成和保存形象均增加本地 ref 级互斥锁；异步回调会校验当前账号 token、当前媒体 ID、当前任务 ID 和当前路由仍属于形象生成链路，避免真机连点、慢网络、切号或重新选图后旧请求误跳页、误覆盖、误保存。不新增页面，不接新接口。
- 普通聊天发送链路状态保护补强：主人聊天页发送和失败重发增加本地互斥锁；本地 sending/failed 气泡会记录所属 `conversationId`，服务器消息合并时只保留当前会话的本地气泡；发送回包会校验当前账号 token 和当前会话，避免切号、切会话或慢网络后旧消息污染新会话。不新增页面，不接新接口。
- 我的页设置业务联动补强：附近可见关闭后会清空本机附近伙伴列表并刷新消息/通知，发现页也会停止自动/手动定位刷新并展示现有空状态；重新开启附近可见或切换模糊定位后会重新定位刷新附近伙伴，避免服务端还停留在旧曝光/旧精度状态。附近列表更新时同步校正当前约遛选中对象。不新增页面。
- 发现页刷新状态收紧：手动下拉/按钮刷新要求本次真实拿到定位才更新附近列表，定位失败不再沿用旧定位制造“已刷新”错觉；关闭附近可见后，即使旧请求晚返回也会清空并拒绝写回附近伙伴列表。不新增页面。
- 通知中心口径纠偏：进入通知页已自动批量置已读并同步后端，文档不再把“通知批量已读”当作待补主链路。
- 权限页进入建档链路卡顿优化：通知权限授权成功、二次登录恢复到已授权通知状态、设置里重新开启通知时，App 不再立即抢前台资源获取 Expo Push Token，而是延迟静默登记；登记前会再次校验当前 token、系统通知权限和 App 内通知开关，关闭通知、退出账号或 token 变化会取消/跳过旧登记任务，避免影响“下一步，添加宠物”的跳转手感。不新增页面。
- 通知设置开启后的设备登记补齐：在设置页把 App 通知从关闭切回开启后，会在服务端设置保存成功、系统通知权限和 App 开关均确认后重新安排 push token 登记；权限快照保存失败时本地也会使用刚读取到的系统权限状态，避免 UI 显示已开启但后端没有设备 token。不新增页面。
- 远程宠物头像加载兜底：`PetAvatar` 在远程图加载前会先显示本地灵伴占位图和 loading 覆盖层，减少 AI 形象确认页、发现页、消息头像出现空白圆框的时间。不新增页面。
- 地点详情点评状态修复：地点点评提交成功后，审核中提示和“再次提交点评”只跟随当前地点自己的 `myPlaceReview`，不再被新增地点页或其他地点的全局审核状态串页影响。不新增页面。
- 地点草稿状态隔离：地点详情页“点评内容”和新增地点页“宠物友好体验”拆成独立草稿与 loading，避免在两个已有页面之间互相带入输入内容或提交中状态。不新增页面。
- 地点动作防重复补强：地图搜索、地点收藏、地点点评和新增地点提交增加本地 ref 锁，减少按钮/键盘连点造成的重复请求窗口；退出账号时同步清理这些保存中状态。不新增页面。
- 普通聊天草稿状态隔离：会话输入框草稿改为按 `conversationId` 保存，切换不同主人会话时不会把上一条未发送内容带到另一个会话；退出账号会清空全部会话草稿。不新增页面。
- 聊天详情会话状态同步：会话列表轮询、手动刷新、标记已读、招呼接受等路径更新 `conversations` 后，当前打开的聊天详情会自动同步最新会话对象，避免列表已变更但详情页仍使用旧的 `canSendMessage/unread/lastMessage` 状态。不新增页面。
- 聊天发送目标保护：聊天详情页不再在当前会话丢失时 fallback 到会话列表第一位；如果会话已失效，会展示现有空状态并引导回消息列表，发送和重发都会提示重新选择，避免用户以为给 A 发消息却误发到 B。不新增页面。
- 地点详情状态同步：地点列表刷新或搜索结果变化后，当前打开的地点详情会同步到最新地点对象；当前地点不在结果里时会清空详情目标并提示回地图重新选择，避免详情页继续使用旧地点数据。不新增页面。
- 地点点评目标保护：地点详情页和点评提交不再在当前地点丢失时 fallback 到地点列表第一位；地点失效时展示现有空状态并引导回地图，避免把点评、收藏或导航操作落到错误地点。不新增页面。
- 约遛目标状态同步：附近列表刷新后，已选约遛对象会同步到最新主人卡片；如果对象不再出现在附近列表，约遛页显示现有空状态并引导回发现页，不再自动 fallback 到列表第一位，避免误发邀请。不新增页面。
