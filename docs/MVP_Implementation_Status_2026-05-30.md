# Lumii MVP 实现状态记录

日期：2026-05-30

## 1. 本轮已实现

- `mobile/App.tsx` 已切换为新的 React Native 原生化 MVP 入口，不再运行旧的 Stitch iframe/文案点击桥接逻辑。
- 新增统一 MVP 类型层：`mobile/src/mvp/types.ts`。

## 2. 当前可演示主链路

- 登录：手机号校验、协议校验、发送中、60s 倒计时、验证码输入、验证码校验、登录成功跳转。
- 权限：定位、照片与相机、通知；Web 预览模拟授权，iOS/Android 使用已有 Expo 权限服务。
- 新用户建档：未添加宠物 -> 宠物基础信息 -> 上传示例照片 -> 识别详情 -> AI mock 生成轮询 -> 结果确认 -> 电子宠物首页。
- 电子宠物：首页健康摘要、体重/疫苗/备忘入口、电子宠物对话 mock 消息流。
- 健康：体重记录、疫苗计划、健康备忘本地 mock CRUD。
- 社交：附近宠物主人列表、猫/狗/想交朋友筛选、打招呼、约遛邀请。
- 消息：会话列表、通知中心、聊天入口。
- 地图：宠物友好地点列表、搜索、公园/医院/猫狗友好筛选、地点详情、点评提交审核中、收藏 mock。
- 我的：多宠管理入口、设置、账号安全、安全中心、退出登录、注销 mock 确认。

## 4. 当前技术边界

- AI 图像生成、AI 对话、地图 POI、社交、健康、消息仍为 mock 数据。
- 真实短信服务已单独验证过，但 MVP 新 mock API 暂不把短信发送地址暴露在 App 内；后续应由后端代理。
- Stitch 继续作为视觉设计源；生产实现目标转为 React Native 原生页面。
- 地图 SDK、对象存储、AI 网关、Push 服务和后端 API Base URL 仍待确认。

## 6. API 门面更新

已新增 `mobile/src/mvp/api.ts`：

- App 页面统一依赖 `lumiiApi`。
- 默认走本地 mock。

- 已支持登录 token 注入到 `Authorization: Bearer <token>`。
- 已补充接口契约草案：`docs/API_Contract_MVP_v0.md`。
