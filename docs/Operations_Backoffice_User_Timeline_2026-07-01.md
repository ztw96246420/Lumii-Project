# 运营后台用户时间线

日期：2026-07-01

## 目标

在“用户管理”里从单个手机号快速还原完整业务轨迹，减少客服、举报复核、处罚申诉和通知排查时在多个菜单之间反复跳转。

## 入口

页面：用户管理。

操作：用户列表每一行新增“时间线”按钮，点击后在列表下方打开用户时间线面板。

接口：`GET /admin/users/{phone}/timeline?kind=all&limit=160`

## 聚合范围

- 账号：注册时间、账号状态、运营备注。
- 宠物：宠物建档、默认宠物状态、物种、品种、生日。
- 宠物日历：体重、疫苗/驱虫、备忘、来源、状态。
- 内容：宠友圈小事、评论、点赞。
- 关系：招呼、约遛、会话关系、关系状态、消息数、通知数。
- 地点：地点点评、新增地点提交。
- AI：图片素材、AI 灵伴生成任务、宠物 AI 对话回复摘要和自动写入动作。
- 安全：用户发起举报、用户被举报、处罚、申诉。
- 客服：App 反馈、客服工单。
- 通知：站内通知入站、已读/未读状态、深链或关联对象。
- 移动端事件：页面、发现、地图、通知点击等已上报埋点。

## 隐私边界

- 会话类记录不展示私信正文，只展示关系、状态、消息数和通知数。
- 宠物 AI 对话只展示后台已有的短摘要、风险标签、自动写入动作，不主动展开完整上下文。
- 内容、备忘、反馈和工单只展示短摘要，前端会截断展示。
- 时间线只开放后台 admin token 调用；不暴露给移动端。

## 筛选

`kind` 支持：

- `all`
- `account`
- `pet`
- `health`
- `content`
- `social`
- `place`
- `ai`
- `safety`
- `support`
- `notification`
- `event`

后端会按白名单兜底，未知 `kind` 自动回到 `all`。

## 验收

- 用户列表可以直接打开时间线面板，不跳出当前页面。
- 默认按事件时间倒序展示，最新事件在前。
- 每条记录包含时间、类型、标题、短摘要、状态、关联对象类型和关联对象 ID。
- 类型筛选只返回对应 `kind` 的记录。
- smoke 覆盖真实 API 链路：创建用户、宠物、小事、招呼、举报、处罚、备忘、反馈、埋点，并验证后台时间线聚合。
