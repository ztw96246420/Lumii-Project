# Product Design Audit: 我的页

日期：2026-06-10  
页面：Lumii / 灵伴「我的」页  
证据截图：`01-profile-current.png`  
审计方式：Product Design combined audit，本地浏览器截图，移动端 viewport 390 × 844。

## 1. Audit Scope

本次只审计单屏「我的」页当前视觉与体验，不覆盖完整设置流、多宠管理流、账号安全流，也不宣称 WCAG 完整合规。

## 2. User Goal And Accessibility Target

用户来到「我的」页时，核心目标是确认当前账号身份、查看当前宠物、进入宠物档案/设置/安全等管理入口。页面需要让用户快速理解“这是我的账号和我的宠物”，并能明确下一步入口。

## 3. Strengths

1. 整体气质已经比早期骨架更接近 Lumii：浅米色背景、白色卡片、柔和橙色、低刺激底部导航，方向是对的。
2. 信息分组清晰：账号区、当前宠物、设置入口三段结构容易扫读。
3. 底部导航当前态明确，「我的」有浅橙背景和图标高亮。
4. 当前宠物卡片里的头像、昵称、品种、年龄体重、标签层级完整，能表达“宠物档案”的核心内容。
5. 设置入口列表的行高、分割线、右侧箭头形成了比较熟悉的移动端模式。

## 4. UX Risks

1. 数据可信度风险较高：这是新测试手机号登录后的页面，但仍展示“奶油的铲屎官”“奶油”“金毛”“3 岁 2 个月”。如果真实用户二次登录或未建档时看到默认宠物，会怀疑账号串了或数据错了。
2. 「当前宠物」与「宠物档案」重复展示同一个“奶油”，但含义不完全清楚。用户可能不知道点当前宠物卡、点宠物档案行、多宠管理分别会去哪里。
3. 顶部账号卡没有明确“编辑资料”或“账号资料”语义。右侧铅笔图标较弱，容易被看成装饰。
4. 「多宠管理 ›」字号和颜色偏轻，作为宠物管理的重要入口，发现性不足。
5. 设置列表里“社交与附近 已开启”“通知设置 已开启”的状态是静态文本，没有说明可点后能修改，也没有给出关闭风险或隐私含义。
6. 页面缺少“未添加宠物”的个人页状态。如果用户没有宠物档案，「当前宠物」区域应该是空状态或添加入口，而不是默认宠物。

## 5. Accessibility Risks

1. 多个图标按钮只有图形，例如设置、编辑、行尾箭头。截图无法确认是否有可访问标签；真机需要检查 TalkBack/VoiceOver 读法。
2. 部分浅色文字和浅色标签可能存在对比度风险，例如「当前宠物」「多宠管理」「已实名 · Lv.3」「疫苗齐全」这类小号浅色文本。
3. 底部导航的点击目标视觉上够大，但图标和文字整体距离底部较近，需真机确认手势区域和安全区是否稳定。
4. 状态标签较小，像「金毛」「疫苗齐全」「活泼」在小屏或系统字体放大时可能换行或挤压。
5. 右侧箭头和编辑图标缺少文字补充，低视力用户可能不容易理解可操作性。

## 6. Opportunity Areas

1. 账号卡可以更像“身份中心”：头像、昵称、手机号、实名状态、编辑入口分层更明确，避免所有信息挤在渐变卡片中。
2. 当前宠物卡可以突出“当前管理对象”，把“多宠管理”放成更强的次级按钮，减少和「宠物档案」行的语义重叠。
3. 设置区可以按功能分组：宠物管理、社交隐私、通知、账号安全。现在一个列表里混在一起，短期够用，但后续扩展会变拥挤。
4. 可以增加真实状态反馈：未添加宠物、定位未授权、通知关闭、附近不可见等，不要全部默认“已开启”。
5. 视觉上可减少顶部渐变卡片里的装饰性大色块，让头像和昵称更稳。现在色块有一点抢信息层级。

## 7. Recommendations

### P0

1. 修正数据状态：没有宠物时不要展示“奶油/金毛”默认档案；显示“还没有添加宠物”并给出“添加宠物”入口。
2. 二次登录后使用真实手机号、真实当前宠物、真实权限/通知状态，避免默认假数据进入「我的」页。
3. 为设置、编辑、当前宠物卡、列表行补可访问标签，至少保证 TalkBack 能读出“编辑个人资料”“进入宠物档案”等语义。

### P1

1. 调整当前宠物区的信息结构：当前宠物卡负责展示当前宠物；「宠物档案」列表行负责进入档案详情，二者文案要避免重复。
2. 强化「多宠管理」入口，可以改成小型胶囊按钮或带加号/切换图标的操作。
3. 设置列表增加分组标题，例如“宠物管理”“隐私与通知”“安全”，为后续扩展留空间。
4. 小号浅色文本做对比度复核，尤其标签和状态文本。

### P2

1. 根据 Figma Make 补齐「我的页-无宠物」「我的页-多宠」「我的页-通知关闭」「我的页-附近不可见」状态。
2. 增加头像/昵称编辑态、手机号换绑态、实名认证状态说明。
3. 后续把黑名单、注销冷静期、隐私协议 WebView 从设置/安全里拆成完整流程。

## 8. Evidence Limits

- 本次只基于一张 Web 预览截图，不能确认真机触摸、TalkBack/VoiceOver、动态字体、暗色模式、加载态和错误态。
- 没有与 Figma Make 源码逐像素对比，所以本报告不判断“是否 1:1 还原”，只判断当前页面体验质量。
- 当前测试账号链路可能带有开发种子数据，数据问题需再结合真实账号状态验证。
