# MVP Development Update 2026-06-11

## 本次进展

- 宠物档案编辑页开始使用当前宠物资料预填表单。
- 编辑宠物时改为调用 `PATCH /pets/:id`，不再误创建新宠物。
- 编辑成功后同步 `activePet`，返回宠物档案页，并让首页/我的页使用最新宠物信息。
- 宠物信息表单补充生日与性别字段。
- 保存按钮增加 loading，避免重复提交。
- 登录会话已支持本地持久化；二次打开 App 会优先恢复会话，服务端返回 401 时清除本地缓存并回到登录。
- Android 系统返回/全屏手势已接入：二级页返回上一页，一级页再按一次退出。
- 宠物首页问候语改为进入首页时轮换，避免长期固定一句话挡住体验。
- “今日小事”发布改为调用 `POST /health/memos` 持久化到健康备忘；发布按钮增加 loading，失败时保留输入内容并提示。
- 快速健康备忘补齐保存失败 toast，避免接口失败时用户没有反馈。
- 宠物档案页“更换”按钮已接入现有上传照片 -> 识别详情 -> AI 生成 -> 形象确认流程，不再只是待接入提示。
- 开始新一轮宠物形象更换前会清空上一轮照片、生成任务和结果预加载状态，避免串到旧图。
- 地点收藏改为持久化接口闭环：`GET /places/favorites`、`PATCH /places/:id/favorite`。
- App 登录后会拉取已收藏地点，点击收藏/取消收藏时乐观更新，接口失败会回滚并提示。
- 地点详情页分享按钮已接入系统分享面板，分享文案包含地点名、地址、距离、评分、宠物友好标签和“来自 Lumii 灵伴”。
- 地点详情页“高德导航”按钮改为打开高德 URI 搜索页，使用地点名 + 地址作为关键词，并在移动端尝试调起高德 App。
- AI 灵伴对话页右上角“灵伴设置”入口已复用已有宠物档案页，不再只是待接入提示。
- 地点点评提交会把用户输入内容发送到测试后端，并通过 `GET /places/reviews/my` 在地点详情页回显自己的审核中点评。
- “新增地点与点评”页面改为提交真实地点草稿：地点名、地址、宠物友好体验会进入 `POST /places/submissions`，不再误提交到默认地点。

## 健康提醒与通知闭环

- 新增测试后端疫苗提醒持久化：`GET /health/vaccine-reminders`、`PATCH /health/vaccine-reminders/:id`。
- App 进入后会拉取当前宠物已开启的疫苗提醒 ID，按钮状态不再只依赖本地内存。
- 点击“开启提醒”会写入服务端；重复点击会提示“这个提醒已经开启”，不再重复生成本地通知。
- 疫苗计划标记完成后，会自动移除该计划提醒。
- `GET /notifications` 会根据已开启提醒生成去重后的健康提醒通知。
- 新增 `POST /notifications/read`，进入通知中心后 App 会自动把未读通知标记为已读，首页/消息页红点会同步消失。
