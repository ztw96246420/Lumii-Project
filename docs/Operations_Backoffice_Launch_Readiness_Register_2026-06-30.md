# 运营后台上线台账与待澄清清单

日期：2026-06-30

## 1. 目的

这份文档承接运营后台 PRD 里的“上线前必须确认的问题”，并和后台新增的 `上线台账` 页面保持同一套口径。

它解决三个问题：

- 不把测试阶段可用误说成生产级完工。
- 把需要业务、合规或运营拍板的问题集中沉淀。
- 明确后台能力和移动端联动关系，避免后台成为割裂系统。

## 2. 后台入口

新增后台一级菜单：

- `上线台账`

新增接口：

- `GET /admin/launch/readiness`

接口返回：

- `summary`：上线口径、P0 待处理数、待澄清问题数、配置联动数。
- `modules`：后台模块成熟度台账。
- `questions`：上线前必须确认的问题。
- `gaps`：生产风险和预留治理项。
- `linkage`：配置中心联动体检里的关注项。

## 3. 页面口径

状态不使用“已完成/未完成”这种粗粒度口径，而是拆成：

- `测试可用`：当前后台和移动端联动已经能支撑真机测试或小范围验证。
- `部分可用`：核心链路已可用，但生产治理能力不完整。
- `生产阻断`：正式上线前必须解决或明确兜底策略。
- `已预留`：后台入口、数据结构或配置项已保留，但不应被误认为上线。

## 4. 当前模块成熟度

第一版上线台账覆盖以下模块：

- 工作台与数据看板。
- 用户排查。
- 宠物档案与宠物日历。
- AI 灵伴生成。
- AI 对话抽检。
- 内容安全任务池。
- 举报与处罚申诉。
- 地点审核。
- 反馈与工单。
- 通知运营。
- 配置中心。
- 数据导出与审计。
- 系统健康与账号权限。

每个模块都记录：

- 后台证据。
- 移动端联动。
- 下一步需要补齐的生产能力。

## 5. 当前 P0 生产阻断

当前仍需重点关注：

- 后台安全：登录失败锁定已接入；MFA、IP 白名单、多管理员仍待补齐。
- 数据底座：当前仍是 JSON state 文件态，不能作为生产级数据库。
- 内容安全：腾讯云文本/图片机审与抽样复审已接入，仍需要确认生产开关、密钥轮换、抽样比例、异步回调和误杀申诉归因机制。
- 图片审核：头像、宠物图、宠友圈图片、封面、工单附件已进入统一图片审核池，仍需要补充长期证据归档和多审核员质检。
- 高风险配置：配置草稿和风险摘要已接入；维护模式、强制更新、全功能关闭等是否需要双人审批仍待确认。
- 合规文本：App 备案、隐私政策、内容审核制度需要生产版确认。

## 6. 待业务持续澄清

后台页面内置以下待澄清问题：

1. 后台正式域名使用 `ops.lumiiapp.cn`、`admin.lumiiapp.cn`，还是先沿用 `/admin`？
2. 生产后台是否必须白名单 IP？
3. 后台账号是否接企业微信、飞书或邮箱 MFA？
4. 内容安全供应商已选腾讯云，是否需要预留阿里云/人工抽检作为灾备？
5. 图片审核失败时，当前策略为 Review 先隐藏等待审核、Block 拒绝或隐藏；是否需要给用户明确申诉入口？
6. 私信是否允许人工查看全文？如果允许，谁审批、保留多久？
7. 用户业务数据清理是否只保留测试环境？
8. AI 失败额度返还规则如何定义？
9. 永久封禁是否必须双人审批？
10. 导出完整手机号是否允许？如允许，谁审批？
11. 地点审核通过是否给用户奖励或贡献者标记？
12. 系统通知是否需要发送审批和频控？
13. 配置强制更新、维护模式、全功能关闭是否必须双人审批？当前已支持先保存草稿再发布。
14. App 备案、隐私政策、内容审核制度是否已准备生产版文本？

## 7. 移动端联动说明

上线台账不会单独维护一份“看起来很完整”的假数据。

它会读取：

- 配置中心联动体检：`adminConfigLinkageItems()`。
- 系统健康：`adminSystemHealth()`。
- 账号权限：`adminAccounts()`。
- 当前后端业务模块的实现证据。

因此后台页面能看到：

- 哪些配置真实影响 `/app/config`。
- 哪些后端接口会按配置强制拦截。
- 哪些能力目前只是预留或后端单侧生效。

## 8. 暂不做的事情

本版本不做：

- 在页面内直接编辑待澄清问题。
- 多管理员分配 owner。
- 双人审批流。
- 自动生成上线结论。

原因：

- 当前用户只需要单 admin 权限。
- 很多问题需要业务策略确认，不能由代码擅自替业务拍板。
- 先把台账和草稿发布治理做成后台可见，后续再接编辑、状态流转和双人审批。

## 9. 后续建议

下一步可继续沿着这张台账推进：

1. 把 P0 问题逐项关闭。
2. 对每个关闭项补审计证据。
3. 为高风险动作接双人审批。
4. 完成腾讯云内容安全生产开关、环境变量、抽样比例、异步回调和误杀申诉归因机制。
5. 为后台账号补 MFA、IP 白名单和多管理员。
6. 把 JSON state 迁移到数据库。
