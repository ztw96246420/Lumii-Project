# 运营后台宠友圈作者处罚闭环

日期：2026-07-02

## 背景

宠友圈后台原本可以隐藏、恢复、删除动态和评论，但如果运营确认作者存在违规行为，还需要跳到用户处罚页手动输入手机号。这个流程容易漏掉证据来源，也不够贴近内容审核场景。

## 本次实现

新增后台接口：

- `POST /admin/social/posts/{postId}/sanction`
- `POST /admin/social/comments/{commentId}/sanction`

新增后台页面能力：

- 宠友圈动态行增加作者处罚快捷动作：
  - 警告
  - 禁言 24h
  - 冻结 72h
- 评论行增加同样的作者处罚快捷动作。
- 点击处罚会要求填写处罚原因。

## 证据与审计

后端会从真实内容对象自动固化处罚证据：

- 动态/评论正文。
- 作者昵称和手机号。
- 内容 ID。
- 内容状态。
- 动态图片 URL，最多 6 张。
- 证据快照 ID 和快照时间。

审计动作：

- `social.post.sanction`
- `social.comment.sanction`

处罚流水字段会记录：

- `source = pet_circle_post` 或 `pet_circle_comment`
- `sourceTargetId`
- `sourceTargetType`
- `evidenceSnapshot`

## 移动端联动

该能力复用已有用户处罚体系，不是独立后台状态。

- 警告：只通知和记录，不限制功能。
- 禁言：限制发布小事、评论、点赞、打招呼、约遛、私信、宠友圈封面、地点投稿、地点点评等互动。
- 冻结/封禁：限制大多数写操作。
- 用户会收到站内通知，点击跳转安全中心。
- 用户仍可通过安全中心提交账号限制申诉。

## 验证

新增烟测：

```bash
node scripts/smoke-social-author-sanction.cjs
```

该脚本会创建真实宠友圈动态和评论，通过后台直接处罚作者，并确认作者后续移动端写操作被后端拦截。

## 暂不做

- 双人审批。
- 批量处罚审批。
- 自动升级处罚策略。
- 处罚命中率复盘。
