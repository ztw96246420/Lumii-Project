# 移动端业务完整性与上线收口审计

日期：2026-07-13

## 1. 审计口径

本轮目标不是只确认页面能打开，而是同时检查：

- 移动端路由是否完整、是否存在不可直达或未渲染页面。
- 移动端操作是否调用真实对应 API，Web 预览是否与服务器数据严格隔离。
- 用户端动作是否能在运营后台形成可处理、可追踪、可审计的闭环。
- 关键正常态、空状态、异常态和恢复动作是否进入自动化回归。
- 正式上线仍依赖哪些生产配置、外部服务和业务签署。

## 2. 本轮已完成

### 2.1 移动端路由完整性

- `AppRoute` 当前共 41 个路由。
- Web 预览路由改为 `Record<AppRoute, true>` 穷举映射；以后新增路由但未加入预览映射时，TypeScript 会直接报错。
- 所有 41 个路由都能进入现有 `renderScreen` 分支。
- 新增直达验收覆盖：灵伴生成中、我发布的小事、备忘、我的反馈、无宠物、上传、识别结果、编辑主人资料、宠物档案、账号安全。

### 2.3 客服工单前后台闭环

自动化已覆盖：

1. 用户提交反馈并生成客服工单。
2. 用户在客服首次回复前补充说明。
3. 后台读取到 `reviewing` 状态并分配客服。
4. 客服回复并解决工单，用户收到站内通知。
5. 用户评价处理结果。
6. 用户重开已解决工单。
7. 后台读取重开次数并再次解决。
8. 用户补充、重开、客服回复、分配和批量操作全部写入审计日志。

移动端 Playwright 同时覆盖：招呼请求举报 -> 我的 -> 设置与隐私 -> 我的反馈 -> 反馈详情 -> 提交补充；真实 HTTP 登录链路改用自动创建的一次性本地后端，不读取旧 IP，也不写入生产服务器测试数据。

- 真实头像缺失或图片加载失败时统一展示中性爪印缺省态。
- 内置金毛资源只保留给明确的 Web 设计夹具和 AI 生成预览，不再作为真实用户兜底数据。
- 新增 `mockAvatar=missing` 专用验收场景，同时断言圆形头像和首页大形象均进入中性缺省态。

### 2.7 附近小事位置锚点与半径治理

- 附近伙伴继续按双方当前定位判断；附近小事改为按查看者当前位置与小事发布时的不可变模糊位置快照判断，发布者旅行或搬家后历史小事不会跟随用户当前位置迁移。
- 小事卡片和移动端接口只返回模糊距离文案，不返回已保存的经纬度；后台详情保留受控位置证据用于审核排障。
- 没有可靠位置快照的历史小事继续保留在本人及已建立关系的宠友圈归档，但不进入附近流，也不使用发布者当前定位推测回填。
- `social.discoverRadiusKm` 收口为 `3km / 5km / 10km` 三档，首发默认 `10km`；运营后台使用下拉档位，服务端对附近伙伴、小事和地图 POI 强制执行后台值，客户端参数不能扩大或缩小范围。
- 修复缺少 `lat/lng` 查询参数时被错误解析为经纬度 `0,0` 的通用问题；无新定位的请求现在沿用账号最近一次有效位置，不再导致附近状态瞬移。

### 2.8 运行中会话失效恢复与宠友圈归档交互

- HTTP 层对带登录凭据请求返回的 `401` 统一触发会话失效恢复；仅当前仍生效的请求 Token 可以触发，旧请求迟到的 `401` 不会误退出新会话，同一 Token 的并发失败只处理一次。
- 移动端收到会话失效后清除本机持久化凭据、内存 Token、当前账号业务状态和提醒任务，回到登录页并明确提示“登录已失效，请重新登录”，不再让用户停留在业务页反复看到加载失败。
- “我发布的小事”补齐可验证的语义操作入口；同一天多条小事保留当天日期并为后续条目显示具体时间和“同日”，每条卡片同时保留完整发布日期时间。
- “我”标记只出现在本人宠友圈头部；已互相接受招呼的他人宠友圈不展示“我”、更换封面或删除入口。
- 三点菜单继续就地展示“查看评论/删除”；删除增加不可恢复二次确认，评论层增加明确关闭按钮，Mock 删除后不再被演示夹具重新生成。

### 2.9 Android Release 权限最小化

- 从正式包主 Manifest 和 Expo 配置移除当前业务未使用的 `RECORD_AUDIO` 与 `SYSTEM_ALERT_WINDOW`；语音录制和系统悬浮层均不是首发功能，不再为未来假设提前索取敏感权限。
- Android 系统备份改为 `allowBackup=false`；账号、宠物和互动数据以服务端为准，本机登录凭据和草稿不进入系统迁移备份面。
- Release 配置门禁会同时检查 Expo 配置和原生主 Manifest，后续预构建或依赖升级重新引入上述权限时直接失败。

### 2.10 疫苗/驱虫计划自助纠错闭环

- 新宠物的疫苗/驱虫计划保持空列表，不再生成任何默认接种或驱虫记录；真实后端回归已锁住该行为。
- 用户可直接修改计划名称和日期，服务端根据日期统一归一为“计划中”或“已逾期”，避免日期与状态冲突。
- 误标完成后可恢复为未完成计划；旧完成通知会清理，提醒不会未经用户确认自动重开。
- 用户可二次确认后删除错误计划；服务端同步清理提醒开关、站内提醒、完成通知和宠物日历事件。
- 真机侧本地提醒同步键纳入计划名称，改名或改日期后会取消旧调度并按最新内容重新安排。
- Mock API、真实 API、RN 页面、移动端核心 HTTP 冒烟与 Playwright 操作链路保持一致。

### 2.11 短信登录失败锁定与后台解锁

- 保留单张验证码最多 5 次错误限制，并新增跨票据的手机号 + 设备/IP 分层失败窗口；默认风险客户端 10 次、账号兜底 20 次后临时锁定 15 分钟。
- 锁定不能通过重新发码绕过；成功登录自动清零，App 按服务端 `availableAt` 展示真实分钟级倒计时。
- 后台用户管理新增“登录锁定”指标、失败来源与解除入口；客服需完成身份核验并填写原因，解锁动作写入独立审计。
- 生产短信模拟测试覆盖 Spug 请求与拒绝回包、随机码、锁定、禁止发码、后台解锁和恢复登录；腾讯云 TC3 作为备用通道保留兼容测试，后台与移动端 Playwright 均保留视觉证据。

### 2.12 账号注销到期永久清理

- 注销短信确认后不再只撤销当前设备，而是写入账号级 token 截止时间并撤销该手机号全部已记录会话；超过会话表保留上限的旧 token 同样立即失效，冷静期内重新短信登录仍可自动撤销。
- 到期清理同时由分钟级调度器和请求前置检查驱动，账号、宠物、宠物日历、AI、社交、聊天、地点贡献、通知、客服及关联运营记录被实际删除。
- 上传原图、灵伴结果图和动效视频进入持久化 COS 销毁队列；失败指数退避重试，不会因为对象存储短暂不可用而恢复业务数据。
- 只保留不含手机号的 HMAC 匿名墓碑，用于阻断注销前旧 token；同手机号到期后重新登录创建全新空账号，不会复活历史数据。
- 后台系统健康新增 `account_deletion_processor`，上线台账新增 `account_lifecycle`；逾期未删除账号会形成 P0 阻断。

### 2.13 地点收藏“想去”闭环

- 地图筛选条新增使用现有视觉语言的“想去”入口，收藏成功文案直接指向该入口，不再提示尚不存在的后续功能。
- 后端新增完整收藏地点列表接口，按收藏顺序返回且不受当前位置、城市或附近半径限制；用户旅行后仍能找回历史收藏。
- 收藏/取消收藏在 ID 列表和完整地点列表中同步乐观更新，失败时双向回滚；空收藏显示明确空状态并可一键回到附近地点。
- 移动端核心 HTTP 与 Playwright 覆盖收藏、集中查看、进入详情和取消后的接口一致性。

### 2.14 生产登录态禁止明文兼容 token

- 生产环境强制拒绝无签名的旧 `lumii-local-手机号` 格式，不能通过环境变量重新开启；仅非生产环境可显式启用旧缓存兼容。
- 正式登录、刷新、退出、全端撤销和注销后阻断统一使用可验签、可过期、可撤销的 `lumii-v1` token；刷新会继承设备标识，普通退出会撤销同设备整条刷新链而不误伤其他设备。
- 后台系统健康新增 `legacy_local_auth`，上线台账新增 `auth_session_security`；生产短信回归直接验证伪造旧 token 返回 401。

### 2.15 生产 AI 禁止测试结果回退

- 生产环境的图片 provider 为 `mock` 或缺少真实供应商密钥时，创建任务直接失败且不消耗额度，不再轮询后返回内置金毛测试图。
- 生产环境拒绝保存 `lumii://` 测试形象；候选图保存接口只接受当前账号已完成任务返回的结果，不能注入任意外部图片。
- 动效 mock 在生产环境会进入明确失败态，不会把静态图片伪装成已生成视频；上线台账 `ai_runtime` 同时检查 GPT Image 2、Seedance（启用时）和 DeepSeek。

### 2.16 附近地点真实坐标闭环

- 新增地点提交前必须获取 10 分钟内的有效定位；定位失败时保留草稿且不先上传图片，后端再以 `PLACE_LOCATION_REQUIRED` / `PLACE_LOCATION_STALE` 做最终拦截。
- 提交记录保存经纬度、精度和捕获时间；后台审核创建 manual 地点时继承这组坐标，后续附近查询始终按地点坐标计算，不会随提交者移动到其他城市。
- 生产环境启动时移除历史 seed 地点；高德缺 Key、超时或无结果时，只返回当前半径内已保存的真实高德/人工地点或空列表，不再回退四条示例地点。
- 无坐标存量地点仍保留在后台治理目录，但不进入用户附近结果；系统健康新增 `place_location_integrity`，上线台账新增 P0 `place_discovery`，生产缺 `AMAP_WEB_SERVICE_KEY` 直接阻断上线。

### 2.17 用户端登录设备管理

- “账号安全”不再展示写死的设备说明，改为读取当前账号真实有效会话；同设备 token 刷新链聚合为一台设备，并明确标记本机。
- 用户可以退出指定其他设备，或一键退出全部其他设备；被撤销设备的当前 token 与同设备历史刷新链立即失效，本机登录态保持有效。
- 用户端只返回设备类型、脱敏尾号和最近活跃时间，不暴露 token、token 摘要、IP、完整设备 ID 或 User-Agent；完整证据继续留在后台受控审计页。
- 移动端覆盖读取中、失败重试、空状态和操作中状态；不再展示尚未实现的邮箱、密码入口，也不再给纯信息行显示误导性箭头。
- 双设备 HTTP 回归覆盖单设备退出、退出其他设备、刷新链、最终退出、401 阻断、后台时间线及业务数据清理。

### 2.18 公网 API 证据与可信代理收口

- 后台上线台账把源站 TLS 与站外真实可达性拆成两项证据；`127.0.0.1 + SNI` 成功只证明 Nginx/证书/源站，不再被误报成公网可用。
- 站外浏览器、真机或监控通过正式域名到达 API 后，会自动形成默认 24 小时有效的外部证据；不保存客户端 IP、Token 或 User-Agent。
- 后端仅信任 `LUMII_TRUSTED_PROXY_IPS` 中的反向代理，默认仅本机 Nginx；真实客户端 IP 优先读取由 Nginx 覆盖的 `X-Real-IP`，伪造 `X-Forwarded-For` 不能绕过短信频控或后台 IP 白名单。
- 新增 `scripts/probe-production-readiness.cjs`，可在生产服务器读取脱敏的 P0、健康、管理员安全和审批策略，不输出管理员凭据、Token 或服务密钥。

### 2.19 生产 Push 验收证据

- 系统健康新增 `expo_push` 检查，区分未启用、未开启 receipt、无真机 token、等待首个成功 receipt、ticket/receipt 失败和已验证成功。
- 脱敏生产探针同步输出有效/失效设备数量与下发、receipt 汇总，不输出完整 Expo Push Token 或用户手机号。
- 当前生产已启用 Expo Push 和 receipt 轮询，但真实设备 token 为 0、成功 receipt 为 0；代码链路与模拟供应商回归通过，仍需在 versionCode 14 真机授权通知后完成首台设备登记和真实送达验收。

### 2.20 今日小事原子发布

- 附近公开小事与宠物日历联动改为单次 `POST /social/pet-circle/posts` 原子提交，由后端在一次状态保存内同时创建小事和关联备忘。
- 不再先写宠物日历、清空草稿后再发布宠友圈；社交发布失败时不会留下孤立备忘，文字和照片草稿继续保留供用户重试。
- 接口返回 `createdMemo`，回归断言同一小事只产生一条 `source=pet_circle` 的关联记录。

### 2.21 协议正文与同意留痕

- 登录页的《用户协议》《隐私政策》和设置页“协议与政策”均读取后端当前正式版本，展示标题、版本、生效日期、说明和完整章节，不再只有不可阅读的勾选文案。
- 短信登录提交 `legalConsentAccepted`；服务端按当时生效的协议/隐私版本、时间、来源和记录 ID 留存，后台用户列表和时间线可查。
- 新增 `LUMII_REQUIRE_LEGAL_CONSENT` 上线开关；开启后未同意请求返回 `LEGAL_CONSENT_REQUIRED`，且不消费已验证 OTP，用户勾选后可以直接重试。
- 开关采用兼容发布顺序：versionCode 13 首次支持同意参数，当前以能准确上报原生构建号的 versionCode 14 作为分发与观测基线；确认旧版活跃量收敛后再启用强制校验。

### 2.22 宠友圈审核结果闭环

- 待人工复核的小事会保留在作者“我发布的小事”中，显示“审核中”；附近列表、直达公开接口及其他用户宠友圈均不可见。
- 后台批准、驳回、恢复或删除后向作者发送强制结果通知；通知点击进入本人归档，不再用公开动态深链打开不可见内容。
- 驳回小事在本人归档显示“未通过”、审核原因和“仅自己可见”，关闭点赞/评论，仅保留删除；紧凑卡和就地菜单已增加横向溢出门禁。

### 2.23 地点贡献历史与点评状态

- “我的”新增“地点贡献”入口和“地点贡献记录”页，长期聚合地点提交与本人点评，展示审核中、已通过、未通过、已隐藏/删除、审核原因、处理时间和贡献分。
- 驳回地点可以把名称、地址和体验原文带回新增地点编辑器；已通过地点及点评可回到对应地点，驳回点评可直接重写。
- 本人点评接口补齐当前地点名；地点详情不再把驳回/隐藏/删除点评当作普通五星点评展示，而是显示真实状态、原因和对应纠错动作。

### 2.24 公网 API 云防火墙与后端监听收口
