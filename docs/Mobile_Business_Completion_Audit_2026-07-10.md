# 移动端业务完整性与上线收口审计

日期：2026-07-11

## 1. 审计口径

本轮目标不是只确认页面能打开，而是同时检查：

- 移动端路由是否完整、是否存在不可直达或未渲染页面。
- 移动端操作是否调用真实对应 API，Web 预览是否与服务器数据严格隔离。
- 用户端动作是否能在运营后台形成可处理、可追踪、可审计的闭环。
- 关键正常态、空状态、异常态和恢复动作是否进入自动化回归。
- 正式上线仍依赖哪些生产配置、外部服务和业务签署。

### 2.3 客服工单前后台闭环

自动化已覆盖：

1. 用户提交反馈并生成客服工单。
2. 用户在客服首次回复前补充说明。
3. 后台读取到 `reviewing` 状态并分配客服。
4. 客服回复并解决工单，用户收到站内通知。
5. 用户评价处理结果。
6. 用户重开已解决工单。
7. 后台读取重开次数并再次解决。
8. 用户补充、重开、客服回复、分配和批量操作全部写入审计日志。

移动端 Playwright 同时覆盖：招呼请求举报 -> 我的 -> 设置与隐私 -> 我的反馈 -> 反馈详情 -> 提交补充；真实 HTTP 登录链路改用自动创建的一次性本地后端，不读取旧 IP，也不写入生产服务器测试数据。

### 2.4 缺失头像不再伪造宠物形象

- 移除首页、聊天、宠友圈、约遛、招呼请求、多宠管理、宠物档案、安全中心等业务页面对内置金毛图片的生产回退。
- 真实头像缺失或图片加载失败时统一展示中性爪印缺省态。
- 内置金毛资源只保留给明确的 Web 设计夹具和 AI 生成预览，不再作为真实用户兜底数据。
- 新增 `mockAvatar=missing` 专用验收场景，同时断言圆形头像和首页大形象均进入中性缺省态。

### 2.7 附近小事位置锚点与半径治理

- 附近伙伴继续按双方当前定位判断；附近小事改为按查看者当前位置与小事发布时的不可变模糊位置快照判断，发布者旅行或搬家后历史小事不会跟随用户当前位置迁移。
- 小事卡片和移动端接口只返回模糊距离文案，不返回已保存的经纬度；后台详情保留受控位置证据用于审核排障。
- 没有可靠位置快照的历史小事继续保留在本人及已建立关系的宠友圈归档，但不进入附近流，也不使用发布者当前定位推测回填。
- `social.discoverRadiusKm` 收口为 `3km / 5km / 10km` 三档，首发默认 `10km`；运营后台使用下拉档位，服务端对附近伙伴、小事和地图 POI 强制执行后台值，客户端参数不能扩大或缩小范围。
- 修复缺少 `lat/lng` 查询参数时被错误解析为经纬度 `0,0` 的通用问题；无新定位的请求现在沿用账号最近一次有效位置，不再导致附近状态瞬移。

### 2.8 运行中会话失效恢复与宠友圈归档交互

- HTTP 层对带登录凭据请求返回的 `401` 统一触发会话失效恢复；仅当前仍生效的请求 Token 可以触发，旧请求迟到的 `401` 不会误退出新会话，同一 Token 的并发失败只处理一次。
- 移动端收到会话失效后清除本机持久化凭据、内存 Token、当前账号业务状态和提醒任务，回到登录页并明确提示“登录已失效，请重新登录”，不再让用户停留在业务页反复看到加载失败。
- “我发布的小事”补齐可验证的语义操作入口；同一天多条小事保留当天日期并为后续条目显示具体时间和“同日”，每条卡片同时保留完整发布日期时间。
- “我”标记只出现在本人宠友圈头部；已互相接受招呼的他人宠友圈不展示“我”、更换封面或删除入口。
- 三点菜单继续就地展示“查看评论/删除”；删除增加不可恢复二次确认，评论层增加明确关闭按钮，Mock 删除后不再被演示夹具重新生成。

### 2.9 Android Release 权限最小化

- 从正式包主 Manifest 和 Expo 配置移除当前业务未使用的 `RECORD_AUDIO` 与 `SYSTEM_ALERT_WINDOW`；语音录制和系统悬浮层均不是首发功能，不再为未来假设提前索取敏感权限。
- Android 系统备份改为 `allowBackup=false`；账号、宠物和互动数据以服务端为准，本机登录凭据和草稿不进入系统迁移备份面。
- Release 配置门禁会同时检查 Expo 配置和原生主 Manifest，后续预构建或依赖升级重新引入上述权限时直接失败。

### 2.10 疫苗/驱虫计划自助纠错闭环

- 新宠物的疫苗/驱虫计划保持空列表，不再生成任何默认接种或驱虫记录；真实后端回归已锁住该行为。
- 用户可直接修改计划名称和日期，服务端根据日期统一归一为“计划中”或“已逾期”，避免日期与状态冲突。
- 误标完成后可恢复为未完成计划；旧完成通知会清理，提醒不会未经用户确认自动重开。
- 用户可二次确认后删除错误计划；服务端同步清理提醒开关、站内提醒、完成通知和宠物日历事件。
- 真机侧本地提醒同步键纳入计划名称，改名或改日期后会取消旧调度并按最新内容重新安排。
- Mock API、真实 API、RN 页面、移动端核心 HTTP 冒烟与 Playwright 操作链路保持一致。
