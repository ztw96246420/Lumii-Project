## 0. 执行原则

1. Stitch 是当前唯一设计源。
2. 前端只接入 Stitch 已存在的页面、状态、弹层和组件变体。
3. 某个完整 UI 如果 Stitch 里没有，前端不自行创造页面。
4. 每次你在 Stitch 补页面后，我先同步页面映射，再补代码。
5. 这份文档是活清单：已补齐的页面会移动到“已同步并接入”，还缺的继续保留在缺口区。
6. Stitch 页面接入前会先做 QA：中文、乱码、导航一致性、页面状态完整性统一记录到 `docs/Stitch_QA_Checklist_v0.md`。
