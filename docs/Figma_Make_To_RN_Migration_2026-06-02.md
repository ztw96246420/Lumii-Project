## 迁移原则

- Figma Make 作为视觉与交互参考源。
- 不再依赖从 Make 复制到 Figma Design 的缺陷版本。
- React Native/Expo 作为真实可运行 App 实现。
- 业务逻辑沿用现有 MVP 状态机与 mock API，视觉从 Make 源码组件和 token 中逐屏转译。
- 优先从源码提取布局、颜色、字号、圆角、阴影和交互状态；截图只作为最终视觉 QA，不再作为主要实现依据。
