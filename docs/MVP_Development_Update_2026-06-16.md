# Lumii MVP 开发记录 2026-06-16

## 本轮补齐：已有页面业务细节与定位兜底

- 时间状态收敛：体重记录弹窗、健康备忘提醒、约遛邀请日期、AI 灵伴对话、普通会话和消息列表不再使用固定 `09:32`、`09:14`、`06-15`、`2026-05-30`、`2026-09-28` 等假时间。
- 体重记录：新增体重时会在打开弹窗时捕获当前时间，保存时把当天 `recordedAt` 传给 API；mock API 和 HTTP API 契约同步支持该字段。
- 健康备忘：新增、编辑、AI 对话自动写入、高风险健康观察写入都会使用当天日期更新 `updatedAt`，便于健康日历聚合。
- 约遛邀请：时间卡片改为按当前日期动态生成今天、明天和后天，避免隔天测试仍显示旧固定日期。
- 定位兜底：发现页刷新和地图定位路径增加业务层超时保护；权限请求 20 秒超时、定位 22 秒超时，失败时释放 loading 并展示现有 toast / 地图定位失败 banner。
- 后端日期一致性：测试后端 `todayIsoDate()` 改为本地日期生成，避免中国时区凌晨被 UTC 日期带到前一天；默认建档体重也改用该日期函数。
- API 文档：`POST /health/weights` 示例补充 `recordedAt`，后续后端替换时按同一契约实现。
- 宠物首页微调：进入 AI 灵伴对话的“有话想和你说”入口从宠物圆形形象舞台内移到圆框下方，避免遮挡灵伴形象。
- 宠物首页微调二次收紧：取消该对话入口的负向上边距，增加与灵伴圆框之间的真实间距，避免不同安卓机型上再次压住头像区域。
- 健康备忘提醒控件：新增健康备忘页里的提醒时间、重复频率、到期前提醒开关不再是静态样式；点击时间会顺延提醒时间，切换重复频率会同步重算提醒时间，关闭提醒后保存时不再传 `reminderAt`。
- 健康备忘接口：mock API、HTTP API 和测试后端同步支持 `reminderAt`、`reminderEnabled`、`repeat`，并对重复频率和提醒时间做基础校验。
- 健康备忘编辑：编辑页已复用提醒时间、重复频率、到期前提醒开关，打开已有备忘会回填原状态，保存时同步更新提醒字段；备忘列表会展示已开启的提醒摘要。
- 体重记录弹窗：新增记录继续展示打开弹窗时捕获的当前时间；编辑已有记录时改为展示记录日期，移除不可点击的日期箭头，避免把历史记录误显示成当前时间。
- 约遛邀请卡片：会话里的约遛邀请不再停留在“后续接入”toast；收到邀请时可直接发送“接受”或“稍后确认”回复，自己发出的邀请只展示已发送状态。
- 会话附件栏：底部“约遛”已接到现有约遛邀请页，“宠物卡”会发送当前宠物资料文本；地点卡片和图片消息仍保留到富消息设计/数据结构补齐后实现，并已同步到缺失清单。
- 消息页刷新：右上角搜索占位恢复为真实刷新按钮，点击会拉取招呼请求、会话和通知，并显示 loading；搜索状态等 Figma 补齐后再做。
- 消息时间收敛：测试后端新生成的招呼、约遛、AI 对话消息使用当前分钟，不再写死 `刚刚`。
