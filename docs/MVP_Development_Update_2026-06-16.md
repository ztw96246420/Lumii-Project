# Lumii MVP 开发记录 2026-06-16

## 本轮补齐：已有页面业务细节与定位兜底

- 时间状态收敛：体重记录弹窗、健康备忘提醒、约遛邀请日期、AI 灵伴对话、普通会话和消息列表不再使用固定 `09:32`、`09:14`、`06-15`、`2026-05-30`、`2026-09-28` 等假时间。
- 体重记录：新增体重时会在打开弹窗时捕获当前时间，保存时把当天 `recordedAt` 传给 API；mock API 和 HTTP API 契约同步支持该字段。
- 健康备忘：新增、编辑、AI 对话自动写入、高风险健康观察写入都会使用当天日期更新 `updatedAt`，便于健康日历聚合。
- 约遛邀请：时间卡片改为按当前日期动态生成今天、明天和后天，避免隔天测试仍显示旧固定日期。
- 定位兜底：发现页刷新和地图定位路径增加业务层超时保护；权限请求 20 秒超时、定位 22 秒超时，失败时释放 loading 并展示现有 toast / 地图定位失败 banner。
- 后端日期一致性：测试后端 `todayIsoDate()` 改为本地日期生成，避免中国时区凌晨被 UTC 日期带到前一天；默认建档体重也改用该日期函数。
- API 文档：`POST /health/weights` 示例补充 `recordedAt`，后续后端替换时按同一契约实现。
- 宠物首页微调：进入 AI 灵伴对话的“有话想和你说”入口从宠物圆形形象舞台内移到圆框下方，避免遮挡灵伴形象。
- 宠物首页微调二次收紧：取消该对话入口的负向上边距，增加与灵伴圆框之间的真实间距，避免不同安卓机型上再次压住头像区域。
- 健康备忘提醒控件：新增健康备忘页里的提醒时间、重复频率、到期前提醒开关不再是静态样式；点击时间会顺延提醒时间，切换重复频率会同步重算提醒时间，关闭提醒后保存时不再传 `reminderAt`。
- 新增健康备忘表单：不再预填“驱虫 / 外用滴剂 · 拜耳拜宠清”等假内容；默认无类型选择，用户需点选备忘类型并填写备注后才能保存。
- 健康备忘接口：mock API、HTTP API 和测试后端同步支持 `reminderAt`、`reminderEnabled`、`repeat`，并对重复频率和提醒时间做基础校验。
- 健康备忘编辑：编辑页已复用提醒时间、重复频率、到期前提醒开关，打开已有备忘会回填原状态，保存时同步更新提醒字段；备忘列表会展示已开启的提醒摘要。
- 体重记录弹窗：新增记录继续展示打开弹窗时捕获的当前时间；编辑已有记录时改为展示记录日期，移除不可点击的日期箭头，避免把历史记录误显示成当前时间。
- 疫苗逾期预约：疫苗页逾期提醒卡右侧“预约”不再只是静态文案，点击会复用现有地图页，自动搜索“宠物医院”并保持医院筛选，方便直接查找附近可处理疫苗/驱虫的地点。
- 新增疫苗计划：疫苗计划页“新增”不再只是 toast；现有页面内展开名称和计划日期表单，支持今天/7 天后/30 天后/90 天后快捷日期，保存走 `POST /health/vaccines`，成功后刷新健康摘要、健康日历和疫苗列表，失败保留草稿。
- 约遛邀请卡片：会话里的约遛邀请不再停留在“后续接入”toast；收到邀请时可直接发送“接受”或“稍后确认”回复，自己发出的邀请只展示已发送状态。
- 约遛草稿：Screen39 的“保存草稿”不再只是 toast；App 会按当前账号手机号 + 对方 ownerId 本地保存地点、时间、留言和保存时间，从发现页或聊天页再次进入同一伙伴约遛页时会自动恢复，约遛邀请发送成功后会清理对应草稿。
- 约遛地点：约遛邀请页不再默认填“滨江绿地”，无草稿时地点为空并要求用户填写；发送成功后会清空地点和留言，避免下次误用旧地点。
- 约遛/招呼话术：约遛留言 placeholder 不再固定写“奶油超爱飞盘”；打招呼快捷话术在没有当前宠物时不再兜底成“2 岁金毛奶油”，避免空档案或切号场景冒出假宠物资料。
- 招呼请求时间：Screen40 的招呼请求卡片不再按列表序号硬写“刚刚 / 2 小时前 / 昨天”；App 会在请求首次出现在本机时记录时间，刷新、接受或忽略后同步清理，展示真实相对时间。
- 通知中心时间：通知缺少后端 `createdAt` 时不再按序号 fallback 成“刚刚 / 2 小时前 / 昨天”；App 会记录通知首次出现在本机的时间并用于分组与展示，后端有 `createdAt` 时仍优先使用后端时间。
- 健康日期展示：健康首页、备忘列表/编辑页、体重历史、删除确认、疫苗计划页和健康日历疫苗提醒卡不再直接展示 `YYYY-MM-DD` 或把日期伪装成当前时分；统一展示“今天 / 昨天 / X 月 X 日”，紧凑日期位使用“今天 / 明天 / 6.23”，只有新增体重弹窗持有真实当前时间时才显示“今天 · HH:mm”。
- 新建宠物表单：默认草稿不再预填“奶油 / 金毛 / 2024-05-30 / 28.4kg”等假资料，改为空表单，仅保留 placeholder 示例；首页“今日体重”也改为只有今天存在体重记录才显示“已记录”。
- 假资料清理：我的页当前宠物卡不再在缺少性格时默认显示“疫苗齐全 / 活泼 / 对小孩友好”，改为基于真实性格、疫苗计划数量、体重和生日完整度生成标签；宠物档案不再按物种硬写体型或按金毛品种硬写“奶油金”毛色。
- 账号状态清理：我的页和账号安全页不再硬写“已实名 · Lv.3”“安全等级高”“上次登录 Android 设备 · 中国大陆”“登录设备 1 台”等无法由当前数据证明的状态；改为基于手机号登录态展示“手机号已验证”、当前运行设备和验证码登录方式。
- 上传/生成兜底清理：识别结果、生成中和形象确认页丢失上传媒体或生成任务时，不再展示 demo 金毛照片或默认灵伴图；改为现有页面内错误态，引导重新选择照片/重新生成。
- 发现页搜索：顶部搜索按钮不再是“后续开放”toast；点击会在现有发现页展开搜索框，可按宠物名、主人昵称、猫狗类型、距离和标签过滤当前 3km 内附近伙伴，空结果清除会同步清搜索词和筛选条件。
- 会话附件栏：底部“约遛”已接到现有约遛邀请页，“宠物卡”会发送当前宠物资料文本，“地点”会把当前选中的宠物友好地点以文字消息发送；发送附件时不再清空当前输入草稿；地点富卡片和图片消息仍保留到富消息设计/数据结构补齐后实现，并已同步到缺失清单。
- 消息页刷新：右上角搜索占位恢复为真实刷新按钮，点击会拉取招呼请求、会话和通知，并显示 loading；搜索状态等 Figma 补齐后再做。
- 今日小事：拍照/相册按钮已接入系统相机和相册选择，最多添加 3 张本地预览图，点已选照片可移除；发布后会把“附照片 X 张”写入健康备忘内容，云端图片存储留待媒体接口补齐。
- 今日小事/地点照片：未选择照片时不再展示示例宠物图或地点点评图，改为空占位格；只有用户真实选择的本地照片才会显示并参与“附照片 X 张”提交摘要。
- 地点新增/点评照片：Screen49 的“添加照片”不再只是 toast，占位照片会在用户选择后切换成本地预览，最多 3 张、点已选照片可移除；提交点评/新增地点时会把“附照片 X 张”写入提交摘要，成功后清空，失败重试保留草稿与照片状态。
- 地点新增/点评表单：新增地点不再预填“云杉宠物友好公园 / 滨江路 88 号”假资料；“自定义”特色标签已在现有表单内展开输入并加入提交内容，点评模式下评分和特色标签也会随点评正文一起提交。
- 地点新增/点评失败草稿：Screen51 的“保存草稿”不再只是关闭弹层和 toast；App 会按当前账号、本次模式和地点 ID 本地保存地点名、地址、正文、评分、特色标签和本地照片预览，重新进入新增地点或同一地点点评时自动恢复，提交成功后清理对应草稿；失败页文案同步改为“当前页面保留，可手动保存”，不再误称已自动保存。
- 地点详情假数据清理：地点详情页不再硬写“官方认证”“1/48”“236 条点评”“06:00 - 22:00”“010-8888-8888”和默认点评正文；缺少真实字段时显示社区共建、封面图、待补充或暂无我的点评，用户提交过点评时用真实内容和真实提交时间回显。
- 地图搜索排序：Screen48 搜索筛选面板里的“距离最近 / 评分最高 / 点评最多”不再是静态假分段；距离、评分和点评数会真实重排当前地点列表，高亮地点同步按排序后的第一项更新；测试后端、mock API 和地点契约已补 `reviewCount`，用户首次点评某地点成功后本地列表会立即递增该地点点评数。
- 地图距离与物种筛选：Screen48 的“距离 3km”不再是静态滑条；点击会在 1km / 3km / 5km 间切换，并按地点距离真实过滤当前地点列表，清空筛选和退出账号会恢复 3km。汪星/喵星友好 chips 已改为真实筛选，优先使用地点 `supportedSpecies` 字段，旧数据缺字段时仅按地点标签/类别做保守推断，不再弹待补字段 toast。
- 宠物首页入口避让：宠物页“XX 好像有话想和你说”的 AI 对话入口下移为圆形灵伴舞台下方的独立卡片，增加头像舞台高度和真实间距，避免在真机上遮挡灵伴形象圆框。
- 地点/反馈时间写入：mock API 和测试后端新建地点点评、新增地点提交、产品反馈、AI 形象保存/反馈不再写死 `刚刚`，统一写入真实 ISO 时间；Screen50/51 的审核进度和草稿卡优先展示接口返回时间。
- 消息时间收敛：测试后端新生成的招呼、约遛、AI 对话消息使用当前分钟，不再写死 `刚刚`。
- 消息列表口径收敛：消息页的 AI 灵伴会话不再把欢迎语伪装成真实最后一条消息；没有真实对话历史时显示待开始。普通会话 `updatedAt` 兼容旧 `HH:mm` 和新 ISO 时间戳，mock API 与测试后端会话更新时间改为写入真实 ISO 时间，发送成功后本地列表立即刷新到当前时间。
- 疫苗新增日期提示：新增疫苗/驱虫计划的日期格式错误提示不再写死 `2026-06-30`，改为按当前日期动态生成示例日期，避免后续测试出现过期示例。
- 地点审核文案收敛：新增地点审核中卡片不再展示“后续真实接口会返回审核单号”等开发占位说明，改为用户可理解的通知中心进度同步文案。
- 聊天消息时间戳：AI 灵伴聊天、普通会话、约遛邀请和招呼接受后的消息 `time` 不再只存 `HH:mm`，前端、mock API 和测试后端统一写入真实 ISO 时间；聊天页日期 chip 兼容历史 `HH:mm` 数据，并按今天/昨天/具体日期友好显示。
- AI 对话失败横幅：宠物对话页网络失败时顶部“查看详情”不再只是 toast；点击会滚动到最新失败消息位置，并提示用户在失败卡片下方重试，和现有失败消息重发入口形成闭环。
- 默认健康种子数据：mock API 和测试后端的新宠物默认体重、健康备忘、疫苗/驱虫计划不再使用固定 `2026-05/06/07` 日期，改为相对当前日期生成，避免健康页和疫苗页在后续测试中出现过期演示数据。
- 验证码页入口收敛：MVP 当前只支持短信验证码，输入验证码页移除“语音验证码”假入口和对应 toast，避免用户点到不可用功能。
- 会话页假入口收敛：普通聊天页底部附件只保留已接逻辑的“地点 / 宠物卡 / 约遛”，移除未接图片存储的“相册”入口；普通会话和招呼请求页右上角只弹 toast 的“更多/设置”入口同步移除。
- 设置页入口收敛：MVP 当前只支持简体中文，设置页移除“语言”行和“语言切换后续开放”toast，避免展示不可操作的固定选项。
- 账号安全入口收敛：账号安全页移除未设计/未接后端的“注销账号”假入口；设置页保留真实可用的“退出登录”，通用占位组件也不再展示“注销账号/危险操作 · 后续开放”。
- 协议/隐私接口文案收敛：`GET /legal/terms` 和 `GET /legal/privacy` 不再返回“MVP 占位版”和 `mvp-placeholder` 版本号，改为面向用户的测试版说明，同时保留正式上线前法务/合规确认的边界。
- 上传图片校验建议收敛：多人多宠、人宠同框、其他动物干扰等 AI 形象生成拦截状态不再提示“后续版本会支持”，改为当前可执行的重拍/裁切/单宠入镜建议。
- 招呼请求举报闭环：Screen40 的“举报”不再只是后续接入 toast；点击会通过现有 `POST /feedback` 以 `safety` 分类提交对象信息，成功后同步忽略该招呼请求并刷新消息入口。
- 安全中心入口收敛：无明确对象的举报/拉黑/黑名单卡片改为信息态，不再弹“后续开放”；“紧急求助指南”复用现有地图能力，直接搜索附近宠物医院和可求助地点。
- 个人资料头像状态清理：编辑资料页选择头像只是在本地打开相册并处理图片，不再显示固定 `62%` 和“上传中”假进度，改为“处理中 / 保存后生效”。
- 资料与宠物档案信息态收敛：手机号说明不再写“后续会在账号安全页开放”；绝育字段当前未进入建档模型，编辑页去掉不可点击箭头，详情页显示“未记录”而不是“待补充”。
- 宠物档案毛色与 AI 对话反馈文案收敛：毛色字段当前没有进入宠物资料模型，详情页显示“未记录”而不是“待识别”；AI 对话负反馈 toast 改为“会作为语气参考”，不再用过度承诺式文案。
