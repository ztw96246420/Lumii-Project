# Lumii Figma Make 缺失页面提示词 2026-06-06

> 规则：以下页面在你补完 Figma Make 设计前，前端不新增正式页面样式。Codex 只做接口、状态机和已有页面逻辑。

## 1. 健康备忘编辑与删除

页面中文名：健康备忘编辑与删除状态

提示词：

请为 Lumii/灵伴移动端 App 生成“健康备忘编辑与删除状态”页面，尺寸为 iPhone 竖屏。整体风格延续 Lumii 当前 Figma Make 源码风格：温暖浅米色背景、白色圆角卡片、柔和橙色主按钮、青绿色健康提示、清晰中文字体层级。页面用于宠物健康管理中的备忘记录编辑。

页面需要包含：
- 顶部返回栏，标题“编辑备忘”。
- 备忘标题输入框，示例“洗澡记录”。
- 多行备忘内容输入框，示例“今天洗澡后耳朵干净，皮肤没有明显泛红。”
- 日期/分类信息行，例如“今天 · 日常护理”。
- 保存按钮，主按钮文案“保存修改”。
- 删除入口，危险色文字或按钮“删除备忘”。
- 删除二次确认弹窗：标题“删除这条备忘？”，说明“删除后不可恢复”，按钮“取消”“确认删除”。
- 保存成功 toast、删除成功 toast、内容为空错误提示。

请同时生成以下状态：默认编辑态、保存中、保存成功 toast、删除确认弹窗、删除后空状态。

## 2. 人工聊天详情与富消息

页面中文名：聊天详情与富消息状态

提示词：

请为 Lumii/灵伴移动端 App 生成“聊天详情与富消息状态”页面，尺寸为 iPhone 竖屏。页面是宠物主人之间的聊天，不是 AI 灵伴对话。视觉风格延续 Lumii：浅米色背景、白色卡片、橙色强调、隐私安全提示轻量展示。

页面需要包含：
- 顶部聊天对象栏：返回按钮、对方宠物头像、名称“林然和奶油”、副标题“模糊距离 · 已互相打招呼”、安全/举报入口图标。

- 图片消息气泡。

- 对方输入中状态。
- 底部输入栏：文本输入、图片按钮、地点按钮、发送按钮。
- 长按消息菜单：复制、删除、举报。

请同时生成：默认聊天、发送中、发送失败、对方输入中、长按菜单、约遛卡片待确认/已接受。

## 3. 发现页搜索筛选与空状态

页面中文名：附近发现搜索筛选与空状态

提示词：

请为 Lumii/灵伴移动端 App 生成“附近发现搜索筛选与空状态”页面，尺寸为 iPhone 竖屏。页面用于搜索附近宠物主人和筛选猫狗主人。保持 Lumii 温暖高级、轻社交、隐私友好的视觉风格。

页面需要包含：
- 顶部标题“发现”，右侧搜索和筛选图标。
- 搜索展开状态：搜索框 placeholder“搜索宠物名、品种、兴趣标签”。
- 筛选 Bottom Sheet：宠物类型（全部/狗狗/猫咪）、距离范围（1km/3km/5km）、互动意愿（想交朋友/可约遛/只线上聊天）、按钮“重置”“查看结果”。
- 筛选结果列表卡片，延续现有附近宠物卡片风格。
- 定位未授权空状态：说明开启定位才能发现附近伙伴，按钮“去开启”。
- 附近为空状态：文案“附近暂时没有新的伙伴”，按钮“扩大范围”。
- 筛选无结果状态：文案“没有符合条件的伙伴”，按钮“清空筛选”。
- 卡片加载 skeleton 状态。

请同时生成：搜索展开、筛选面板、定位未授权、附近为空、筛选无结果、加载中。

## 4. 地点点评审核与异常状态

页面中文名：地点点评审核与异常状态

提示词：

请为 Lumii/灵伴移动端 App 生成“地点点评审核与异常状态”页面，尺寸为 iPhone 竖屏。页面属于宠物友好地点模块，用于新增地点、提交点评后的状态反馈。延续 Lumii 地图页的温暖白色底部 sheet、橙色主行动、轻量地图元素。

- 收藏/取消收藏 toast 状态。
- 分享地点 Bottom Sheet：分享到聊天、复制链接、系统分享。
- 外部导航确认弹窗：高德地图优先，按钮“取消”“打开高德地图”。

请同时生成：默认提交、审核中、重复地点、提交失败、分享面板、导航确认弹窗。

## 5. 黑名单管理与解除拉黑

页面中文名：黑名单管理与解除拉黑

提示词：

请为 Lumii/灵伴移动端 App 生成“黑名单管理与解除拉黑”页面，尺寸为 iPhone 竖屏。页面属于设置与安全中心。视觉风格保持克制、安全、清晰，避免沉重压迫感。

页面需要包含：
- 顶部返回栏，标题“黑名单管理”。

- 空状态：文案“暂无黑名单用户”。
- 操作成功 toast：已解除拉黑。

请同时生成：列表状态、空状态、解除确认弹窗、解除成功 toast。
