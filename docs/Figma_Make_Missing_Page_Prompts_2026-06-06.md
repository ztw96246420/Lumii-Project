# Lumii Figma Make 缺失页面与状态提示词

维护日期：2026-06-12

## 使用规则

- 这份文档只记录“还没有设计源、不能由前端自行发挥”的页面、弹层、复杂状态和关键交互。
- 已经有 React Native 路由或基础实现的页面，不再列为“缺失页面”；如果只是缺某个细分状态，会明确写成“缺失状态/弹层/组件”。
- 新页面、新 Bottom Sheet、新复杂弹窗、新卡片样式、新富消息样式，必须先由用户在 Figma Make 补齐设计，再由 Codex 转译实现。
- 当前开发优先级：先做已有页面内的逻辑、状态联动、接口契约、缓存、Bug 修复；尽量暂缓新增页面和外部模型/API。
- 视觉源优先级：Figma Make `Lumii Project Opus4.7` > 源码包 > 截图比对 > Codex 临时 scaffold。

以下内容不要再误认为“缺失页面”。后续如果要增强，只按“缺失状态/弹层/组件”记录。

### 0. 健康日历总览页（暂缓开发）

类型：新页面。当前健康管理页、健康备忘页、体重页、疫苗计划页已有基础实现；这里不重复要求这些页面，只记录尚未原生化的“健康日历总览”独立页面。

什么时候需要：用户希望按日期查看体重、疫苗/驱虫、健康备忘等所有健康事件时。

接口状态：`GET /health/calendar` 已有 MVP 测试后端和 mock API 支持，会按当前宠物聚合体重、疫苗和健康备忘；这里只缺 Figma Make 的独立日历页设计。按当前开发优先级，暂缓开发。

Figma Make 提示词：

```text
请为 Lumii/灵伴移动端 App 生成“健康日历总览页”，尺寸为 iPhone 竖屏。注意：健康管理页、体重页、疫苗计划页、健康备忘页已经存在，不要重新设计这些基础页面；本次只设计按日期聚合健康事件的日历总览。

视觉风格严格延续 Lumii Project Opus4.7：浅暖背景、白色柔和卡片、真实卡通化宠物头像、青绿色健康提示、橙棕色主按钮、圆润但克制，所有文案必须为简体中文。

页面需要包含：
1. 顶部返回栏，标题“健康日历”。
2. 当前宠物小卡：宠物头像、昵称、健康分或最近状态。
3. 月份切换区域：当前月份、上一月/下一月图标按钮。
4. 日历网格：每一天用小圆点或轻量标签显示当天是否有体重、疫苗、备忘事件。
5. 选中日期状态：高亮选中日期，并在下方展示该日事件列表。
6. 事件列表卡片：体重记录、疫苗/驱虫计划、健康备忘三种类型要有不同但克制的图标或色点。
7. 空状态：选中日期无事件时显示“这一天还没有健康记录”。
8. 即将到期提醒：疫苗/驱虫临近或逾期时，用温和提示，不制造医疗焦虑。
9. loading、读取失败、下拉刷新、重试按钮状态。

请同时生成默认月视图、选中日期有事件、选中日期为空、疫苗即将到期/逾期、加载中、读取失败六种状态。
```
