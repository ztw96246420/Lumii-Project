# Lumii 设计门禁规则 2026-06-06

## 核心原则

新增页面、弹窗、Bottom Sheet、重要空状态、复杂表单和关键交互状态，不允许先由前端自行设计样式。

后续流程必须是：

1. Codex 先输出 Figma Make 页面提示词和状态清单。
2. 用户在 Figma Make 补齐设计。
3. Codex 从 Figma Make/源码包/截图对照拉取或转译。
4. Codex 只实现业务逻辑和设计还原，不自行发挥视觉风格。
5. 如必须临时补逻辑，页面只能作为无视觉承诺的隐藏/开发态 scaffold，不能进入用户验收主链路。

## 允许先做的内容

- 数据类型、mock API、状态机、接口契约。
- 已有页面内的非视觉逻辑修复。
- 已有组件的 bug 修复。
- 不新增页面的路由保护、表单校验、倒计时、toast 逻辑。

## 必须先等设计的内容

- 新独立页面。
- 新弹窗、确认框、Bottom Sheet。
- 新卡片样式、列表样式、复杂表单布局。
- 聊天富消息样式。
- 地图地点、审核、分享、收藏等新增状态页。
- AI 形象反馈、重生成、多候选等新增体验页。

## 当前需补 Figma Make 的页面

- 健康备忘列表/新增/编辑/删除确认。
- 人工聊天详情页。
- 聊天富消息：图片、地点卡片、约遛卡片、发送失败、对方输入中、长按菜单。
- 地点新增/点评/审核中/重复地点/提交失败。
- 发现页搜索、筛选、无结果、定位未授权、附近为空。
- 黑名单管理、解除拉黑、注销冷静期。
