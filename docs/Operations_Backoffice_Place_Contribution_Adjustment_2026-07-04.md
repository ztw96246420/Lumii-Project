# 运营后台：地点贡献纠偏

日期：2026-07-04

## 背景

地点贡献分此前已经覆盖自动入账和用户本人公开身份：

- 新增地点审核通过：`+10` 贡献分。
- 关联已有地点：`+5` 贡献分。
- 后台配置可控制移动端“地点共建者”身份是否展示，以及最低展示门槛。

但运营仍缺少纠偏能力。误审、刷分、线下活动补分、客服争议处理都不能靠手改状态文件解决，否则会破坏审计链路，也无法保证移动端重新计算一致。

## 已实现

### 后台地点页

“地点贡献者”卡片新增：

- `手动调整`
  - 输入用户手机号。
  - 输入调整分值，支持正数补分、负数扣分。
  - 可选关联地点 ID。
  - 必填调整原因。
- 单条贡献记录 `撤销`
  - 不物理删除记录。
  - 写入 `voidedAt`、`voidedBy`、`voidedReason`。
  - 撤销后不再计入用户地点共建身份。
- 策略标识
  - 移动端身份：已联动。
  - 排行榜：预留。
  - 活动奖励 / 兑换：预留。

### 后端接口

- `GET /admin/places/contributions`
  - 汇总只统计 `active` 贡献记录。
  - 返回 `manualAdjustments` 和 `voided`，用于后台识别纠偏量和撤销量。
- `POST /admin/places/contributions/adjust`
  - 创建 `manual_adjustment` 贡献记录。
  - `points` 范围：`-1000` 到 `1000`，且不能为 `0`。
  - 需要已注册手机号和 4 字以上原因。
  - 写入审计：`place.contribution.adjust`。
  - 给用户写入系统通知。
- `POST /admin/places/contributions/:id/void`
  - 将有效贡献记录标记为 `voided`。
  - 写入审计：`place.contribution.void`。
  - 给用户写入系统通知。

### 移动端联动

`/me.placeContributionSummary` 现在只按有效记录计算：

- 自动贡献和手动调整都会影响 `points`、`total`、`level`、`publicEligible`。
- 已撤销记录保留在后台账本，但不再计入移动端身份。
- 用户侧 `points` 不展示负数；如果运营扣分超过历史贡献，移动端按 `0` 处理，原始净值保留在 `rawPoints` 便于后台/客服排查。

### 数据原则

- 贡献分仍是运营积分账本，不等同现金、余额、虚拟资产或可提现权益。
- 撤销是状态变更，不是物理删除，保证审计和争议处理可追溯。
- 手动调整适用于纠偏和活动补记，不代表活动奖励、兑换、排行榜已经上线。

## 回归验证

`scripts/smoke-place-contributions.cjs` 已扩展覆盖：

1. 新增地点审核通过记录 `+10`。
2. 关联已有地点记录 `+5`。
3. 后台汇总为 `15` 分。
4. 后台手动调整 `+7`。
5. `/me.placeContributionSummary` 重算为 `22` 分。
6. 后台撤销这条手动调整。
7. 后台汇总和 `/me.placeContributionSummary` 回到 `15` 分。
8. 贡献身份配置仍通过 `/app/config` 控制移动端展示。

## 仍需确认

- 是否要做排行榜：全站榜、城市榜、附近榜，还是活动榜。
- 贡献分是否接活动奖励；如果接，需要活动预算、发放规则、反作弊和申诉流程。
- 是否开放兑换；如果开放，需要权益库存、财务口径、有效期和撤销策略。
- 他人主页是否展示贡献身份；当前仍只展示给用户本人。
